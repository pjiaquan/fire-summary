Shipping Rust Core v2 in a Browser Extension

We moved article extraction, ranking, and prompt packaging into Rust so the browser extension could keep a deterministic content pipeline.

The biggest benefit was not raw speed. It was being able to test the extraction and classification logic with stable fixtures instead of only clicking through the popup manually.

What changed

The extension now extracts structured blocks, scores them, applies a token budget, and sends a compressed context payload to Gemini instead of pushing the entire page text into the model.

That makes long-form writing easier to summarize because the important narrative blocks are preserved while boilerplate and repeated navigation text get dropped earlier in the pipeline.

Tradeoffs

Build and release flows became more complex, so we added diagnostics, fixture regression, and CI checks to keep the Rust core maintainable.
